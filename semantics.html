Semantic HTML and Page Meaning

One big thing I noticed from studying the template is how important semantic HTML is. Tags like <header>, <main>, <section>, and <footer> do more than just hold content. They tell the browser and assistive technologies what each part of the page is supposed to represent.

In my own pages, I tried to follow this idea by wrapping the main content in <main>, keeping the site title and menu inside <header>, and ending every page with a simple <footer>. This makes the structure more predictable and easier to follow when someone reads the HTML directly.

Using semantic tags also helps with accessibility and search engines, even if my project is small. A clear hierarchy of headings and sections makes it easier for screen readers to move through the page. It also helps me as a learner, because I can quickly see where each part of the site starts and ends.
